# SpatialUnderstandingDataProcess

## 项目概述

**空间理解多模态 VLM 训练数据处理平台** - 支持双角色权限、数据获取、AI 辅助可视化与转换

---

## 用户画像

### 目标用户：大模型研究员

| 维度 | 特征 |
|------|------|
| **技术背景** | 有大模型理论基础，懂 Transformer、VLM 等概念 |
| **编程能力** | 会写 Python 脚本，熟悉常见数据格式（JSON/Parquet/CSV） |
| **数据处理能力** | 能独立编写处理脚本，但不想重复造轮子 |
| **工作场景** | 快速准备训练数据，格式转换，质量检查 |
| **痛点** | 数据处理流程繁琐、格式不统一、重复劳动多 |
| **偏好** | 效率优先，不喜欢过度封装，能接受一定学习成本 |

### 设计理念

1. **低码开发（Low-Code）** - 类似 Simulink，图形化拖动执行逻辑，减少手写代码
2. **AI 辅助编程** - 研究员写的代码越少，AI 生成的代码越多 → 越成功
3. **数据不出本地** - 敏感数据不传给 AI，只传递流程描述和脱敏错误信息
4. **效率优先** - 一键操作 > 图形化配置 > 手写代码
5. **可解释性** - 告诉用户"为什么这么做"，而非"点这个按钮就行"
6. **灵活性** - 允许自定义，而非"只能这样用"

### 数据安全边界

**可以传给 AI 的**：
- ✅ 用户描述（自然语言）
- ✅ 流程结构（节点类型、连接关系）
- ✅ 代码模板/框架（不含实际数据）
- ✅ 错误信息（脱敏后）
- ✅ 公开的技术文档

**不能传给 AI 的**：
- ❌ 实际数据内容
- ❌ 数据格式结构（公司内部规范）
- ❌ 文件路径/命名规则
- ❌ 业务敏感信息

## 设计要点

### 用户系统
- **双角色权限**：超级用户（全功能）+ 普通用户（仅浏览/导出）
- **禁用公开注册**：仅支持超级用户手动创建账户
- **权限隔离**：用户只能访问自己的数据

### LLM 配置管理（新增）
- **统一配置**：LLM Base URL、API Key、Model Name 仅超级用户可配置
- **数据库存储**：配置加密存储，AI 服务从数据库读取
- **普通用户不可见**：隐藏所有 LLM 配置入口和 AI 功能

### AI 功能定位（重构后）
- **人在环（Human-in-the-Loop）**：AI 生成代码模板 → 人在本地调整 → 本地执行
- **数据安全优先**：敏感数据不出本地，仅传递脱敏信息给 AI
- **低码流程设计器**：图形化拖动节点设计流程，AI 生成执行代码
- **使用场景**：
  - 流程设计器（节点拖动 + 代码生成）
  - 错误诊断助手（脱敏错误信息 + AI 建议）
  - 代码模板库（预置常见操作模板）
  - 质量规则生成（AI 辅助设计检查规则）

### 流程设计器（新增核心功能）
- **节点库**：预置 20+ 常用处理节点（数据加载、提取、转换、增强、导出、质检）
- **画布**：拖放节点、连接流程、参数配置
- **代码生成**：流程 JSON → AI 生成 Python 代码 → 本地执行
- **执行监控**：实时日志、错误定位、中间结果预览
- **AI 协作点**：节点推荐、代码生成、错误诊断（均不接触实际数据）

### 存储设计
- T 级数据，可插拔存储层，路径可配置

### 训练目标
- 多模态 VLM 训练

### 数据格式
- 用户提供目标格式，不定义标准输出
- 支持图像 - 文本、视频 - 文本配对数据（暂不支持点云/网格）

### 可视化
- **通用卡片式展示（内置保底）**：自动识别 `image` 字段，卡片网格布局，每页 50 个，最多 30 页
- **图片处理**：支持 URL/Base64/路径，自动下载到 `数据集/images/`
- **视频处理**：提取第一帧作为封面，显示"不支持播放"
- **多张图片**：卡片内按顺序全部显示
- **AI 自定义可视化**：特殊需求时 AI 动态生成 HTML（可选）

### 其他
- 搜索：按名称搜索（初期）
- 标注：暂不提供标注功能
- 质量检查：AI 辅助分析
- 备份：增量备份 + 定时任务 + 大文件优化

### US-024: 流程设计器 - 低码数据处理（新增核心功能）

**描述**：作为研究员，我可以通过图形化拖动节点来设计数据处理流程，无需手写代码，AI 根据流程生成执行代码

**设计理念**：类似 Simulink 的模块化开发，流程 = 大模块

**数据安全原则**：
- 实际数据不出本地
- 仅传递流程结构（节点类型、连接关系）给 AI
- AI 生成代码模板，本地替换路径后执行

---

#### 核心概念（2026-03-10 讨论确定）

##### 1. 模块类型

| 类型 | 说明 | 端口特征 |
|------|------|----------|
| **功能模块** | 处理数据的核心模块 | 有输入端口 + 输出端口 + 可变参数 |
| **分支模块** | 条件分流，数据透传，必有兜底分支 | 多个输出端口（按条件激活） |

**取消/延后**：
- ~~输入适配器~~ → 变成流程输入端口
- ~~输出适配器~~ → 变成流程输出端口
- ~~数据中继器~~ → 优先级极低，暂不考虑

##### 2. 端口概念

- 每个模块有明确的**输入端口**和**输出端口**
- 每个端口 = 一种数据类型
- 端口数量由模块功能决定
- 用户描述模块时，AI 解析并定义端口

**示例**：
```
抽帧模块
┌─────────────────────┐
│     video ──► [IN]  │  ← 输入端口：视频数据
│                     │
│     fps = 1.0 (参数) │  ← 可变参数：抽帧率
│                     │
│ [OUT] ──► frames    │  ← 输出端口：图像序列
└─────────────────────┘
```

##### 3. 流程端口（悬空端口）

- 流程本身也有输入端口和输出端口
- 悬空端口不隶属于任何模块
- 用户拖入画布，连接到起点/终点模块
- **自动推断**：连接到模块输入端口 → 该模块成为起点

```
[流程输入端口] ──► [起点模块] ──► ... ──► [终点模块] ──► [流程输出端口]
```

##### 4. 连线校验规则

| 规则 | 说明 |
|------|------|
| 唯一起点模块 | 多个流程输入端口可连到同一个起点模块 |
| 唯一终点模块 | 多个流程输出端口可来自同一个终点模块 |
| 违规处理 | 拒绝连线 + 提示错误 |

##### 5. 分支模块

```
分支模块（数据透传，仅决定流向）
┌─────────────────────┐
│  data ──► [IN]      │
│         │           │
│         ├───────► [OUT1] 条件A满足 → 激活
│         │           │
│         ├───────► [OUT2] 条件B满足 → 激活
│         │           │
│         └───────► [OUT3] default（兜底）→ 激活
│                     │
│  所有输出 = 输入数据  │
└─────────────────────┘
```

**特点**：
- 不处理数据，数据透传（输出 = 输入）
- 仅根据条件选择激活哪个输出端口
- 必须有兜底分支（固定名为 default）
- 条件优先级：按表单顺序，可拖动调整

#### 模块创建流程

##### 功能模块
```
用户拖入功能模块 → 双击打开创建面板
├─ 输入端口定义（名称 + 描述）
├─ 输出端口定义（名称 + 描述）
├─ 可变参数定义
└─ 功能描述框
    → 提交 → 后端解析 → OpenClaw → AI生成代码
```

##### 分支模块
```
用户拖入分支模块 → 双击打开创建面板
├─ 输入数据定义（名称 + 描述，让AI知道数据结构）
├─ 条件分支定义（可拖动排序）
│   ├─ 条件1描述
│   ├─ 条件2描述
│   └─ [＋ 新增条件]
├─ 兜底分支（置灰，固定名为 default，不可删除）
└─ 提交 → 后端解析 → OpenClaw → AI生成代码
```

用户双击已创建的模块 → 显示模块基础信息 + 反馈框
├─ 模块名称、端口、参数、功能描述

#### 模块复用策略

**问题**：模块在不同流程中复用时，编辑操作如何不影响其他流程？

**方案**：实例化 + 可选同步
- 编辑默认只影响当前流程（安全优先）
- 提供显式选项让用户选择是否全局同步
- 避免意外破坏其他流程

---

#### 流程存储

```
流程目录/
├── workflow.json    # 配置（模块、端口、连线）- 前端展示/编辑用
└── workflow.py      # 生成的执行代码（拓扑排序后的模块调用）- 后端运行用
```

**触发更新的时机**：
| 场景 | 处理 |
|------|------|
| 创建新流程 | 生成 workflow.json + workflow.py |
| 编辑流程（连线变化） | 更新 workflow.json + 重新生成 workflow.py |
| 编辑模块（模块代码变化） | 只更新模块代码，workflow.py 可选重新生成 |
