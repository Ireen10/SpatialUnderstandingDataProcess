# SpatialUnderstandingDataProcess

## 项目概述

**空间理解多模态 VLM 训练数据处理平台** - 支持双角色权限、数据获取、AI 辅助可视化与转换

---

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| 后端 | Python 3.10+ / FastAPI |
| 前端 | React 18+ / TypeScript / Vite |
| 数据库 | PostgreSQL 15+ (生产) / SQLite (开发) |
| 缓存 | Redis 7+ |
| 存储 | 本地存储 + S3 兼容接口（MinIO/AWS S3/阿里云 OSS） |
| 任务队列 | Celery 5+ / Redis |
| LLM | OpenRouter (z-ai/glm-5)，仅超级用户可配置 |
| 部署 | Docker / Docker Compose |

---

## 设计要点

### 用户系统
- **双角色权限**：超级用户（全功能）+ 普通用户（仅浏览/导出）
- **禁用公开注册**：仅支持超级用户手动创建账户
- **权限隔离**：用户只能访问自己的数据

### LLM 配置管理（新增）
- **统一配置**：LLM Base URL、API Key、Model Name 仅超级用户可配置
- **数据库存储**：配置加密存储，AI 服务从数据库读取
- **普通用户不可见**：隐藏所有 LLM 配置入口和 AI 功能

### AI 功能定位
- **自动化流程**：AI 生成代码 → 自动保存 → 直接使用
- **无需人工审查**：信任超级用户的输入，无需额外审核流程
- **使用场景**：可视化代码生成、转换脚本生成、数据质量分析

### 存储设计
- T 级数据，可插拔存储层，路径可配置

### 训练目标
- 多模态 VLM 训练

### 数据格式
- 用户提供目标格式，不定义标准输出
- 支持图像 - 文本、视频 - 文本配对数据（暂不支持点云/网格）

### 可视化
- AI 根据数据类型动态生成 HTML 可视化页面

### 其他
- 搜索：按名称搜索（初期）
- 标注：暂不提供标注功能
- 质量检查：AI 辅助分析
- 备份：增量备份 + 定时任务 + 大文件优化

## 部署方式

### 一键部署（推荐）

```bash

# Linux/Mac
chmod +x deploy.sh
./deploy.sh
```

### Docker Compose

```bash
cp .env.example .env
docker compose up -d --build
```

访问地址：`http://<你的 IP>`
